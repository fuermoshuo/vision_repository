# 相机标定 逐行详解

> 适用读者：对计算机视觉和相机模型不熟悉的初学者。

---

## 一、先搞清楚：相机标定在干什么？

### 你可能遇到的问题

用相机拍照时，实际拍出来的照片和现实世界并不完全一致：

- **畸变**：一条直线在照片里变弯了（像透过鱼眼镜看东西）
- **比例不对**：照片里的 100 像素，到底对应现实中的几厘米？

**相机标定**就是解决这两个问题的：算出相机的"内参"（焦距、光心）和"畸变系数"，之后就能把照片里的像素坐标换算成真实世界坐标。

### 一个类比

```
你的眼睛 = 相机
    │
    ├── 晶状体 = 镜头（决定焦距、畸变）
    ├── 视网膜 = 图像传感器（把光变成电信号）
    └── 大脑   = 标定程序（算出晶状体的参数）
```

标定 = 给相机做一次"验光"，搞清楚它的"度数"。

## 二、前置知识：针孔相机模型

### 三维世界 → 二维照片

想象一个针孔相机的侧面：

```
                                    图像平面（照片）
    3D世界点 P(X, Y, Z)                │
         ●                            │
          \                           │  p(x, y) ← 光线打在照片上的位置
           \                          │   ●
            \                         │
             \   针孔（光圈）          │
              ●───────────────────────│
              ↑ 光心                   │
           焦距 f
```

这就是**针孔模型**：世界中的一个点，光线穿过光心，投影到图像平面上。

### 数学模型

现实世界中的一个点 $(X, Y, Z)$ 投影到照片上的像素 $(u, v)$，公式是：

$$
\begin{bmatrix} u \\ v \\ 1 \end{bmatrix}
=
\underbrace{
\begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}
}_{\text{相机矩阵 } K}
\cdot
\begin{bmatrix} X/Z \\ Y/Z \\ 1 \end{bmatrix}
$$

| 符号 | 含义 | 大白话 |
|------|------|--------|
| $f_x, f_y$ | x 和 y 方向的焦距（像素单位） | 镜头"有多长"，单位是像素 |
| $c_x, c_y$ | 光心坐标（像素单位） | 照片的"正中心"在哪，通常约等于分辨率的一半 |
| $K$ | **相机矩阵 / 内参矩阵** | 这 4 个数完整描述了相机的投影规律 |

### 畸变模型

但现实中的镜头不是完美的针孔，图像会有畸变：

```
理想位置                                   实际位置（桶形畸变）
    ●                                         ●
  ●   ●                                   ●     ●
●   ·   ●                               ●   ·   ●
  ●   ●                                   ●     ●
    ●                                         ●
    
（网格是方的）                            （网格被拉扯弯了）
```

这个项目用**径向畸变**（只考虑前 3 个系数 $k_1, k_2, k_3$）和**切向畸变**（$p_1, p_2$）：

| 系数 | 名称 | 说明 |
|------|------|------|
| $k_1, k_2, k_3$ | 径向畸变系数 | 光线穿过镜头边缘时弯得更多，越靠边变形越大 |
| $p_1, p_2$ | 切向畸变系数 | 镜头和传感器不平行导致的变形 |

本项目 `distort_coeffs` 存储的就是 $[k_1, k_2, p_1, p_2, k_3]$。

## 三、标定板是什么？

这个项目用的是**非对称圆点标定板**（Asymmetric Circles Grid）：

```
○  ○  ○  ○  ○  ○  ○  ○  ○  ○      ← 10列
  ○  ○  ○  ○  ○  ○  ○  ○  ○  ○
○  ○  ○  ○  ○  ○  ○  ○  ○  ○      ← 偶数行偏移半格
  ○  ○  ○  ○  ○  ○  ○  ○  ○  ○
○  ○  ○  ○  ○  ○  ○  ○  ○  ○
  ○  ○  ○  ○  ○  ○  ○  ○  ○  ○
○  ○  ○  ○  ○  ○  ○  ○  ○  ○      ← 7行

相邻圆心距 = 30mm（center_distance_mm）
```

**为什么用圆点而不是棋盘格？**
- 圆心的检测精度比棋盘格角点更高
- 非对称排列让 OpenCV 能自动区分标定板的朝向（不会上下颠倒搞混）

### 第 19-31 行：`centers_3d()` —— 生成标定板 3D 点

```cpp
std::vector<cv::Point3f> centers_3d(const cv::Size &pattern_size, float center_distance)
{
    std::vector<cv::Point3f> centers_3d;
    for (int i = 0; i < pattern_size.height; i++)       // 遍历每一行
    {
        for (int j = 0; j < pattern_size.width; j++)    // 遍历每一列
        {
            centers_3d.push_back({j * center_distance, i * center_distance, 0});
            //                    ↑ X坐标(毫米)      ↑ Y坐标(毫米)       ↑ Z=0
        }
    }
    return centers_3d;
}
```

| 参数 | 示例值 | 含义 |
|------|--------|------|
| `pattern_size` | (10, 7) | 标定板有 10 列、7 行 |
| `center_distance` | 30.0 | 相邻圆心的距离（毫米） |
| 返回值 | `[{0,0,0}, {30,0,0}, {60,0,0} ... {270,180,0}]` | 70 个 3D 坐标 |

**关键假设**：标定板放在 Z=0 的平面上（世界坐标系原点在第一个圆的圆心），所有圆心共面。

{
    bool       save_requested = false;    // 用户按了 's' 键 → 保存当前帧
    bool       quit_requested = false;    // 用户按了 ESC → 退出
    std::mutex mutex;                     // 互斥锁，保护多线程访问

```cpp
bool sdl_event_callback(const SDL_Event &event, void *user_data)
{
    auto *state = static_cast<CalibrationState *>(user_data);

    if (event.type == SDL_KEYDOWN)
    {
        std::lock_guard<std::mutex> lock(state->mutex);
        switch (event.key.keysym.sym)
        {
        case SDLK_s:       state->save_requested = true;  return true;
        case SDLK_ESCAPE:  state->quit_requested = true;  return true;
        }
    }
    return false;
}
```

| 概念 | 解释 |
|------|------|
| `std::lock_guard` | 自动上锁/解锁，"{}块内持有锁，离开自动释放" |
| `static_cast<T *>` | 安全的指针类型转换 |
| `void *user_data` | SDL 的通用指针，什么都能传 |

**为什么要多组数据？**
一组数据像"一张照片里的一个视角"，OpenCV 需要从**多个不同角度**拍到的标定板来联立方程求解。至少需要 10 组（项目中 `min_images = 10`）。

### 第 183-196 行：执行标定 —— `calibrateCamera()`

```cpp
    if (saved_images < min_images)
    {
        MAS_LOG_ERROR("Not enough images, need at least {}", min_images);
        return -1;
    }

    // 设置终止条件
    auto criteria = cv::TermCriteria(
        cv::TermCriteria::COUNT + cv::TermCriteria::EPS,  // 条件：迭代次数够了 OR 精度够了
        100,          // 最多迭代 100 次
        DBL_EPSILON); // 精度要求：变化小于机器精度就停止

    // 核心函数！
    cv::calibrateCamera(
        obj_points,      // 输入：所有图片的 3D 点集合
        img_points,      // 输入：所有图片的 2D 点集合
        img_size,        // 输入：图像分辨率
        camera_matrix,   // 输出：相机内参矩阵（3×3）
        distort_coeffs,  // 输出：畸变系数（5个）
        rvecs,           // 输出：每张图的旋转向量
        tvecs,           // 输出：每张图的平移向量
        cv::CALIB_FIX_K3,// 标志：固定 k3=0（只用 k1, k2, p1, p2）
        criteria);       // 终止条件
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj_points` | `vector<vector<Point3f>>` | 10 组，每组 70 个 3D 坐标 |
| `img_points` | `vector<vector<Point2f>>` | 10 组，每组 70 个像素坐标 |
| `camera_matrix` | `Mat(3,3,CV_64F)` | 输出的 3×3 相机内参矩阵 |
| `distort_coeffs` | `Mat(1,5,CV_64F)` | 输出的 5 个畸变系数 |
| `rvecs, tvecs` | 输出 | 每张图拍摄时标定板的位姿（旋转+平移） |
| `CALIB_FIX_K3` | 标志位 | 固定第 3 个径向畸变系数为 0，防止过度拟合 |

**`calibrateCamera` 内部在做什么？**

```
1. 建方程：对每一对 (3D点 → 2D点)，列出投影方程
       3D点 × 相机矩阵 × 畸变模型 = 2D点

2. 求初始解：忽略畸变，用直接线性变换（DLT）估计相机矩阵

3. 迭代优化（Levenberg-Marquardt）：
       不断调整 K 和畸变系数，让重投影误差最小
       直到"迭代100次"或"改进微乎其微"
```

### 第 198-214 行：计算重投影误差

```cpp
    double error_sum    = 0;
    size_t total_points = 0;
    for (size_t i = 0; i < obj_points.size(); i++)
    {
        std::vector<cv::Point2f> reprojected_points;
        // 用标定好的参数，把 3D 点投影回 2D
        cv::projectPoints(
            obj_points[i],   // 3D 坐标
            rvecs[i],        // 该图的旋转
            tvecs[i],        // 该图的平移
            camera_matrix,   // 刚算好的相机矩阵
            distort_coeffs,  // 刚算好的畸变系数
            reprojected_points);  // 输出：投影出来的 2D 点

        total_points += reprojected_points.size();
        for (size_t j = 0; j < reprojected_points.size(); j++)
        {
            // 计算"实际检测到的位置"和"投影出来的位置"之间的距离
            error_sum += cv::norm(img_points[i][j] - reprojected_points[j]);
        }
    }
    double error = error_sum / total_points;  // 平均每个点的误差
```

| 概念 | 含义 |
|------|------|
| **重投影** | 用算出来的参数，把 3D 点"投回去"看看落在哪 |
| **重投影误差** | 投回去的位置 与 实际检测到的位置 之间的平均像素距离 |
| 好的标定 | 误差 < 0.3 像素 |
| 差的标定 | 误差 > 1 像素 → 数据有问题，要重采 |

## 六、关键概念速查表

| 概念 | 简单解释 | 类比 |
|------|---------|------|
| **内参矩阵 $K$** | 描述相机内部光学特性的 3×3 矩阵 | 眼睛的"度数" |
| **外参 $R, t$** | 描述相机在世界中的位置和朝向 | "你站在哪里、朝哪看" |
| **径向畸变** | 光线穿过镜头边缘时弯得更厉害 | 哈哈镜效果 |
| **切向畸变** | 镜头和传感器不平行导致的变形 | 照片一边大一边小 |
| **重投影误差** | 用算好的参数反推位置，和实际位置差多少 | "按公式算出来应该在 (100,100)，实际检测在 (100.2, 99.8)" |
| **标定板** | 已知几何尺寸的参考图案 | 一把刻度精确的尺子 |
| **PnP** | Perspective-n-Point：从 3D-2D 对应点对求解相机位姿 | "给我几个已知点，我告诉你相机在哪" |
| **`CALIB_FIX_K3`** | 固定第 3 个径向畸变系数为 0 | 简化模型，防止过拟合 |
| **`findCirclesGrid`** | OpenCV 函数，检测圆点标定板 | 自动找圆心的工具 |
| **`SPSCQueue`** | 单生产者单消费者队列 | 传送带：一头放，一头取 |

## 八、标定结果的用途

标定完后，`camera_matrix` 和 `distort_coeffs` 被用在：

1. **去畸变**：`cv::undistort(img, undistorted, K, distort_coeffs)` — 把弯的图片拉直
2. **PnP 位姿解算**：已知内参，就能从 2D 图像点反推 3D 位姿（手眼标定中用到）
3. **像素 → 物理坐标转换**：目标检测时，把像素坐标映射到实际物理坐标

```
原始图像（有畸变）──→ undistort() ──→ 矫正图像（直的）
直线上墙的边缘是弯的                   墙的边缘是直的 ✓
```
