# 手眼标定 逐行详解

> 适用读者：对多坐标系变换、IMU、四元数不熟悉的初学者。

## 一、先搞清楚：手眼标定在解决什么问题？

### 一个生活场景

你在一个房间里，有一张地图（世界坐标系）。你站在某个位置，手里拿着一个 GPS 定位器（IMU），头上戴着一个 GoPro（相机）。

现在你拍了张照片，照片里有一把椅子。

**问题是：这把椅子在地图上的什么位置？**

```
已知：
  ├── GPS 告诉你："你面朝北，站在 (10, 5)"  → 你自己的位置（IMU 数据）
  ├── GoPro 拍到了椅子                        → 椅子在照片里的像素位置
  └── 已知椅子的 3D 模型                      → 标定板的已知尺寸

未知：
  └── GoPro 装在你头上的角度和位置             → 相机和云台之间的相对位姿
```

**手眼标定**就是算出"GoPro 和你脑袋之间的相对位姿"——即**相机和云台之间的旋转平移关系**。

### 为什么叫"手眼"？

| 类比 | 机器人 | 本项目 |
|------|--------|--------|
| 👁 **眼** | 装在机械臂末端的相机 | 装在云台上的相机 |
| ✋ **手** | 机械臂末端执行器 | 云台（Gimbal） |
| 🔧 **标定** | 算出手和眼之间的固定位姿 | 算出云台和相机之间的固定位姿 |

```
         "手" (云台/Gimbal)
              │
        ┌─────┴─────┐
        │            │
    固定但未知    已知（IMU 提供）
    的位姿关系     云台在世界中的朝向
        │
        ▼
         "眼" (相机/Camera)
              │
              ▼
         看到标定板 →
              │
         已知板在
         相机坐标系下的位姿
         （通过 PnP 求解）
```

## 二、坐标系关系全景图

这个项目涉及 **6 个坐标系**，一层套一层：

```
┌─────────────────────────────────────────────────────────┐
│                    坐标系关系图                           │
│                                                         │
│                         IMU_Absolute                     │
│                         (惯性测量单元绝对坐标系)            │
│  ┌─────────────────────────┐                            │
│  │      R_imubody2imuabs    │ ← IMU 四元数实时提供         │
│  │         (实时变化)        │                            │
│  │                          ▼                            │
│  │                     IMU_Body                         │
│  │                   (IMU 本体坐标系)                     │
│  │  ┌─────────────────────────┐                         │
│  │  │   R_gimbal2imubody       │ ← 配置文件中的固定值       │
│  │  │      (固定不变)           │                         │
│  │  │                          ▼                         │
│  │  │                      Gimbal                        │
│  │  │                   (云台坐标系) ← "手"                │
│  │  │  ┌─────────────────────────┐                      │
│  │  │  │  R_gimbal2camera         │ ← ★ 这就是我们要算的！  │
│  │  │  │  t_gimbal2camera         │    "手眼关系"         │
│  │  │  │      (固定，未知→待求)     │                      │
│  │  │  │                          ▼                      │
│  │  │  │                      Camera                       │
│  │  │  │                   (相机坐标系) ← "眼"              │
│  │  │  │  ┌─────────────────────────┐                    │
│  │  │  │  │  R_board2camera          │ ← PnP 实时求解     │
│  │  │  │  │  t_board2camera          │                    │
│  │  │  │  │      (每帧不同)           │                    │
│  │  │  │  │                          ▼                    │
│  │  │  │  │                      Board                    │
│  │  │  │  │                   (标定板坐标系)               │
│  │  │  │  └─────────────────────────┘                    │
│  │  │  └─────────────────────────┘                       │
│  │  └─────────────────────────┘                          │
│  └─────────────────────────┘                             │
└─────────────────────────────────────────────────────────┘
```

| 坐标系 | 全称 | 说明 |
|--------|------|------|
| **World** | 世界坐标系 | 固定的大地参考系 |
| **Board** | 标定板坐标系 | 以标定板第一个圆点为原点，Z 轴垂直板面向外 |
| **Camera** | 相机坐标系 | 以相机光心为原点，Z 轴沿光轴方向 |
| **Gimbal** | 云台坐标系 | 云台旋转中心 |
| **IMU_Body** | IMU 本体坐标系 | IMU 传感器的物理安装方向 |
| **IMU_Absolute** | IMU 绝对坐标系 | IMU 的"水平面"参考（类似东北天坐标系） |

### 第 27-44 行：`centers_3d()` —— 生成标定板 3D 坐标

```cpp
std::vector<cv::Point3f> centers_3d(const cv::Size &pattern_size, float center_distance_mm)
{
    std::vector<cv::Point3f> centers_3d;
    for (int i = 0; i < pattern_size.height; i++)
    {
        for (int j = 0; j < pattern_size.width; j++)
        {
            float x = 0;   // ★ 注意：所有 X 坐标都是 0！
            float y = (-j + 0.5 * pattern_size.width)  * center_distance_mm;
            float z = (-i + 0.5 * pattern_size.height) * center_distance_mm;
            centers_3d.push_back({x, y, z});
        }
    }
    return centers_3d;
}
```

**和相机标定版的区别**：

| 项目 | 相机标定 | 手眼标定 |
|------|---------|---------|
| X 轴 | `j * center_distance` | **全部为 0** |
| Y 轴 | `i * center_distance` | `(-j + 5) * 30`（居中） |
| Z 轴 | **全部为 0** | `(-i + 3.5) * 30`（居中） |
| 标定板平面 | XY 平面（平放在桌上） | **YZ 平面**（竖直挂起） |

```
相机标定时的标定板（平放）：         手眼标定时的标定板（竖直）：
        ┌───────┐                      侧视图 →      ┌──┐
        │       │  ← Z=0               Camera →     │板│
        │ ○ ○ ○ │                      看过来 →     │子│
        │ ○ ○ ○ │                                   └──┘
        └───────┘                             YZ 平面，X=0
    XY 平面（水平）
```

**为什么手眼标定时 X=0？**
因为标定板竖直挂起，其法向量朝向 X（相机看向标定板的方向），所有圆心在 YZ 平面上。

**关键依赖链**：手眼标定**依赖**相机标定的结果（`camera_matrix`、`distort_coeffs`），执行顺序不能反。

```cpp
    // ③ 读取云台 → IMU 的固定旋转关系（硬件安装时就确定的）
    auto            R_gimbal2imubody_data = config["handeye_calibration"]["R_gimbal2imubody"].as<std::vector<double>>();
    Eigen::Matrix3d R_gimbal2imubody = Eigen::Matrix<double, 3, 3, Eigen::RowMajor>(R_gimbal2imubody_data.data());
```

`R_gimbal2imubody` 是硬件装配时就确定的常量：

```yaml
handeye_calibration:
  R_gimbal2imubody: [0, 1, 0, -1, 0, 0, 0, 0, 1]
```

这个 3×3 矩阵的含义：

$$
R_{gimbal2imubody} = \begin{bmatrix}
0 & 1 & 0 \\
-1 & 0 & 0 \\
0 & 0 & 1
\end{bmatrix}
$$

表示云台坐标系绕 Z 轴旋转了 90° 就是 IMU 本体坐标系的朝向。这是物理安装角度决定的，出厂就固定了。

### 第 132-251 行：主循环 —— 核心逻辑

#### 步骤①：获取 IMU 四元数，计算云台朝向

```cpp
        // 获取 IMU 四元数
        Eigen::Quaterniond q = user_serial.q(frame_time);
```

串口收到 IMU 传来的**四元数** $(w, x, y, z)$，它描述 IMU 本体在绝对空间中的姿态。

| 概念 | 简单解释 |
|------|---------|
| **四元数** | 用 4 个数表示 3D 旋转，比欧拉角好（不会万向节死锁） |
| **欧拉角** | 用 3 个角度（yaw/pitch/roll）表示旋转，直观但会死锁 |

**为什么用四元数不用欧拉角？**
欧拉角有个致命缺陷叫"万向节死锁"——当 pitch=90° 时，yaw 和 roll 变成同一个轴，你无法区分它们。四元数没有这个问题。

```cpp
        // 四元数 → 旋转矩阵
        Eigen::Matrix3d R_imubody2imuabs = q.toRotationMatrix();
```

```
四元数 (w, x, y, z)  ──toRotationMatrix()──→  3×3 旋转矩阵
     │                                                  │
     │  4 个数                                        9 个数
     │  紧凑、无死锁                                   直观、好算
```

```cpp
        // 🔑 关键变换：IMU 本体 → 云台 → 世界
        Eigen::Matrix3d R_gimbal2world =
            R_gimbal2imubody.transpose()   // ① 先 "脱掉" IMU 的外壳
            * R_imubody2imuabs             // ② 用 IMU 数据得到绝对朝向
            * R_gimbal2imubody;            // ③ 再 "穿回" 云台坐标系
```

**为什么是这个公式？**

一步一步看：

```
目的：求云台在世界中的朝向 R_gimbal2world

已知：
  R_gimbal2imubody  →  云台 → IMU本体（硬件固定，配置里给的）
  R_imubody2imuabs  →  IMU本体 → IMU绝对（IMU实时给的）

过程：
  R_gimbal2world = R_imuabs2world × R_imubody2imuabs × R_gimbal2imubody

  但 R_imuabs2world 我们假设 = I（IMU绝对坐标系就是世界坐标系）
  而且这是一个相似变换：
  R_gimbal2world = (R_gimbal2imubody)^T × R_imubody2imuabs × R_gimbal2imubody
                     ↑                        ↑                      ↑
                R_imubody2gimbal          IMU的绝对旋转           云台→IMU本体

用线性代数的话说：先转到IMU本体坐标系下做旋转，再转回云台坐标系。
```

通俗类比：

```
你站在一个旋转台上。
旋转台（云台）上面装了个陀螺仪（IMU）。
陀螺仪告诉你："我现在面朝北偏东30°"。
但你真正想知道的是："旋转台朝哪个方向？"

因为陀螺仪是斜着装在旋转台上的（不是完全对齐），
你需要：
  1. 先算出陀螺仪和旋转台的相对关系
  2. 用陀螺仪的读数反推旋转台的朝向
```

```cpp
        // 旋转矩阵 → 欧拉角（用于屏幕显示）
        Eigen::Vector3d ypr = rm_utils::eulers(R_gimbal2world, 2, 1, 0) * 57.3;
```

`eulers(R, 2, 1, 0)` → 按 Z→Y→X 顺序分解旋转矩阵，得到 yaw、pitch、roll（弧度）。`* 57.3` 转成角度。

#### 步骤②：检测标定板 + PnP 求解

```cpp

        bool board_detected = cv::findCirclesGrid(image, pattern_size, centers_2d);
```

和相机标定一样，用 `findCirclesGrid` 找圆心。

```cpp

        if (should_save && board_detected)
        {
            // ★ PnP：从 2D 像素坐标 + 已知 3D 模型 + 相机内参 → 解出标定板在相机中的位姿
            cv::Mat rvec, tvec;

            cv::solvePnP(
                centers_3d_,      // 3D 点（标定板圆心坐标，世界单位）
                centers_2d,       // 2D 点（检测到的像素坐标）
                camera_matrix,    // 相机内参（来自相机标定）
                distort_coeffs,   // 畸变系数（来自相机标定）
                rvec,             // 输出：旋转向量 → 板在相机坐标系下的朝向
                tvec,             // 输出：平移向量 → 板在相机坐标系下的位置
                false,            // 不使用扩展参数
                cv::SOLVEPNP_IPPE // ★ 特殊算法：针对平面物体的 PnP
            );
```

| 参数 | 含义 |
|------|------|
| `centers_3d_` | 标定板圆心的 3D 坐标（世界单位），`centers_3d()` 生成的 70 个点 |
| `centers_2d` | 图像中检测到的圆心像素坐标，也是 70 个点 |
| `camera_matrix` | 相机内参（从相机标定得到的） |
| `distort_coeffs` | 畸变系数 |
| `rvec, tvec` | 输出：标定板在相机坐标系下的位姿 |
| `SOLVEPNP_IPPE` | **Infinitesimal Plane-Based Pose Estimation**：专门为平面上的点设计的 PnP 算法，对共面点更稳定 |

**PnP 在做什么？**

```
已知：
  ├── 标定板圆心在"板子坐标系"中的 3D 坐标
  ├── 这些圆心在照片中的 2D 像素坐标
  └── 相机的内参（焦距、光心、畸变）

求解：
  └── 板子坐标系相对于相机坐标系的旋转和平移
      （也就是"相机前面多远、朝什么方向有一块板子"）
```

```
                相机光心
                   ●
                  /|\
                 / | \
                /  |  \
               /   |   \    ← 光线
              /    |    \
             ●     ●     ●  ← 标定板上的圆点（在照片上各有一个像素位置）
```

```cpp
            // 计算 R_world2gimbal（云台在世界中的位姿的逆）
            Eigen::Matrix3d R_world2gimbal = R_gimbal2world.transpose();
            cv::Mat         R_world2gimbal_cv;
            cv::eigen2cv(R_world2gimbal, R_world2gimbal_cv);
            cv::Mat t_world2gimbal = (cv::Mat_<double>(3, 1) << 0, 0, 0);
            // ↑ 平移设为 0 —— 因为我们只关心朝向，不在乎云台的位置
```

**为什么 `t_world2gimbal = (0, 0, 0)`？**

因为 IMU 只提供朝向（旋转），不提供位置（平移）。在这个场景下，云台的位置是固定的（相对于世界原点），我们只关心相机相对于云台的旋转——这正是手眼标定的核心。

```cpp
            // 保存这组配对数据
            rvecs.emplace_back(rvec);                          // PnP：板→相机
            tvecs.emplace_back(tvec);
            R_world2gimbal_list.emplace_back(R_world2gimbal_cv); // IMU：世界→云台
            t_world2gimbal_list.emplace_back(t_world2gimbal);    // 全部 (0,0,0)
        }
```

**每按一次 's' 保存的数据**：

```
采集到的一组数据 = {
    板→相机的旋转和平移（PnP 算的）
    世界→云台的旋转（IMU 算的，平移恒为0）
}
```

采集多组（至少 3 组）后，不同姿态构成联立方程组，就能解出"云台→相机"的固定关系。

### 第 253-278 行：执行手眼标定

```cpp

    {
        MAS_LOG_ERROR("Calibration failed, need at least 3 data, got {}", rvecs.size());
```

**为什么至少 3 组？** 手眼标定的基本方程 $AX = XB$ 中，未知数有 6 个（3 旋转 + 3 平移），每组数据提供一组约束，数学上至少需要 3 组不同位姿。

```cpp
    cv::Mat R_gimbal2camera, t_gimbal2camera;  // ★ 核心结果：云台→相机
    cv::Mat R_world2board,    t_world2board;    // 附带结果：世界→标定板

    cv::calibrateRobotWorldHandEye(
        rvecs,                  // 输入：标定板在相机中的旋转（PnP 结果）
        tvecs,                  // 输入：标定板在相机中的平移（PnP 结果）
        R_world2gimbal_list,    // 输入：云台在世界中的旋转（IMU 结果）
        t_world2gimbal_list,    // 输入：云台在世界中的平移（全 0）
        R_world2board,          // 输出：标定板在世界中的旋转
        t_world2board,          // 输出：标定板在世界中的平移
        R_gimbal2camera,        // ★ 输出：相机在云台中的旋转 ← 我们要的！
        t_gimbal2camera);       // ★ 输出：相机在云台中的平移 ← 我们要的！
```

**`calibrateRobotWorldHandEye` 在解什么方程？**

这是 OpenCV 提供的"机器人-世界-手-眼"联合标定函数。它求解的问题是：

$$
A_i \cdot X = Z \cdot B_i
$$

对于每一组数据 $i$：

| 符号 | 含义 | 本项目对应的量 |
|------|------|--------------|
| $A_i$ | 物体在相机中的位姿（每帧不同） | `rvecs[i]`, `tvecs[i]`（PnP 求的） |
| $B_i$ | 机器人在世界中的位姿（每帧不同） | `R_world2gimbal[i]`, `t_world2gimbal[i]`（IMU 求的） |
| $X$ | **相机在机器人中的位姿**（固定，要求解的） | `R_gimbal2camera`, `t_gimbal2camera` |
| $Z$ | 物体在世界中的位姿（固定，附带求解的） | `R_world2board`, `t_world2board` |

图解：

```
        世界 (World)
          │    \
   Z=A₀X=ZB₀│     \ Z=A₂X=ZB₂
          │        \
       云台₀       云台₂
       (B₀)        (B₂)
        │            │
        X (固定!)    X (固定!)
        │            │
      相机₀        相机₂
        │            │
       A₀           A₂
        │            │
        └── 标定板 ──┘
           (固定不动)
```

**关键**：标定板不动，相机+云台一起动。从不同角度拍摄同一块板，联立方程分离出 $X$（手眼关系）和 $Z$（板的位置）。

```cpp
    t_gimbal2camera /= 1e3;  // mm → m（OpenCV 输出单位是毫米）
    t_world2board    /= 1e3;  // mm → m
```

### 第 282-305 行：计算派生变换和偏角

```cpp
    // ① 算逆变换：相机→云台（上面的结果是云台→相机）
    cv::Mat R_camera2gimbal, t_camera2gimbal;
    cv::transpose(R_gimbal2camera, R_camera2gimbal);   // 旋转矩阵的逆 = 转置
    t_camera2gimbal = -R_camera2gimbal * t_gimbal2camera;  // 平移的逆 = -R^T·t

    // ② 算逆变换：标定板→世界
    cv::Mat R_board2world, t_board2world;
    cv::transpose(R_world2board, R_board2world);
    t_board2world = -R_board2world * t_world2board;
```

**为什么旋转矩阵的逆 = 转置？**
旋转矩阵是**正交矩阵**，它的逆等于它的转置。这是 3D 几何学的基本性质，不用求逆矩阵（省计算）。

### 第 292-305 行：计算相机偏离理想安装角

```cpp
    // ③ 相机同"理想情况"的偏角
    Eigen::Matrix3d R_camera2gimbal_eigen;
    cv::cv2eigen(R_camera2gimbal, R_camera2gimbal_eigen);  // OpenCV Mat → Eigen Matrix

    // 理想的云台安装：相机朝向正前方
    Eigen::Matrix3d R_gimbal2ideal{{0, -1, 0},   // X轴（gimbal）→ Y轴（ideal）
                                   {0, 0, -1},   // Y轴（gimbal）→ Z轴（ideal）
                                   {1, 0, 0}};  // Z轴（gimbal）→ X轴（ideal）

    Eigen::Matrix3d R_camera2ideal = R_gimbal2ideal * R_camera2gimbal_eigen;
    Eigen::Vector3d camera_ypr     = rm_utils::eulers(R_camera2ideal, 1, 0, 2) * 57.3;
```

**`R_gimbal2ideal` 是什么？**

约定的"理想安装姿态"：相机光轴正对前方。这个矩阵定义了"如果装得完美，云台坐标系和理想相机坐标系之间应该怎么变换"。

实际安装一定会有偏差（几度的偏角），这个计算就是告诉你 **"你的相机装歪了多少"**。后续算法可以拿这个角度做补偿。

```cpp
    // ④ 标定板到世界坐标原点的水平距离
    auto x        = t_board2world.at<double>(0);
    auto y        = t_board2world.at<double>(1);
    auto distance = std::sqrt(x * x + y * y);  // 水平面上的距离（忽略高度）
```

```cpp
    // ⑤ 标定板同竖直摆放时的偏角
    Eigen::Matrix3d R_board2world_eigen;
    cv::cv2eigen(R_board2world, R_board2world_eigen);
    Eigen::Vector3d board_ypr = rm_utils::eulers(R_board2world_eigen, 2, 1, 0) * 57.3;
```

如果标定板挂得绝对竖直，pitch 和 roll 应该是 0°。实际偏角告诉你标定板挂歪了多少。

## 五、数据采集的操作技巧

```bash
./mas_vision calibrate_handeye
```

采集时需要**移动相机（连云台一起）**从不同角度拍摄固定不动的标定板：

```
         俯视图：标定板
              │
     位置1 ◄──┼──► 位置3     ← 相机在位置1、2、3各拍一张
              │               （云台也随之转动）
     位置2 ◄──┘

每次按 's' 保存：确保检测到标定板，且 IMU 数据稳定
按 'c' 清空：如果某次保存的数据有问题
按 'q' 退出：触发计算
```

采集技巧：

| 要点 | 原因 |
|------|------|
| 标定板不动，相机动 | 手眼标定的前提 |
| 不同的朝向（yaw 变化要大） | 给算法提供丰富的角度约束 |
| 至少采集 5-10 组 | 3 组是最小值，多点更稳定 |
| 保持标定板在画面中 | 否则 `findCirclesGrid` 检测不到 |
| IMU 稳定后再按 's' | 避免运动中的 IMU 噪声 |

## 七、关键概念速查表

| 概念 | 简单解释 | 在本项目中的作用 |
|------|---------|----------------|
| **四元数** | 用 4 个数表示 3D 旋转 | IMU 数据传输姿态 |
| **旋转矩阵** | 3×3 矩阵表示旋转 | 做坐标变换 |
| **欧拉角 (YPR)** | 3 个角度：Yaw/Pitch/Roll | 给人看的（直观） |
| **PnP** | 从 3D-2D 对应点求解位姿 | 算出标定板在相机中的位置 |
| **手眼标定** | 求相机和机器人之间的固定变换 | 求 R_camera2gimbal |
| **`calibrateRobotWorldHandEye`** | OpenCV 的联合标定函数 | 一次算出手眼关系 + 板的位置 |
| **正交矩阵** | 旋转矩阵是正交矩阵，$R^{-1} = R^T$ | 求逆旋转直接转置 |
| **`R_gimbal2ideal`** | 相机"理想安装姿态" | 和实际姿态比较得出偏角 |
| **`SOLVEPNP_IPPE`** | 专门给平面上的点用的 PnP | 标定板是平面的，用这个更稳 |
| **`cv::eigen2cv`** | Eigen 矩阵 ↔ OpenCV Mat | 在两个数学库之间转换 |

## 八、和相机标定的区别总结

| 维度 | 相机标定 | 手眼标定 |
|------|---------|---------|
| **目的** | 求相机内参 $K$ 和畸变系数 | 求相机和云台的固定位姿关系 |
| **输入** | 标定板的 2D-3D 对应点 | PnP 结果 + IMU 姿态 |
| **标定板** | 平放在桌上（XY 平面） | 竖直挂起（YZ 平面） |
| **相机** | 固定，标定板移动 | 标定板固定，相机移动 |
| **核心函数** | `calibrateCamera()` | `calibrateRobotWorldHandEye()` |
| **预处理** | 无 | **必须先做相机标定** |
| **最少数据量** | 10 组 | 3 组 |
| **输出** | `camera_matrix` + `distort_coeffs` | `R_camera2gimbal` + `t_camera2gimbal` |
| **保存位置** | `config/hikcamera.yaml` | 同一个文件，`handeye_calibration` 段 |

## 九、一个完整的坐标系变换链示例

假设相机看到了一个点，想知道它在世界坐标系中的位置：

```
像素坐标 (u, v)
    │  去畸变 + 内参反投影
    ▼
相机坐标系中的 3D 点 P_cam
    │  × R_camera2gimbal + t_camera2gimbal  ← ★ 手眼标定结果
    ▼
云台坐标系中的 3D 点 P_gimbal
    │  × R_gimbal2world ← IMU 实时数据
    ▼
世界坐标系中的 3D 点 P_world  ← 最终想要的！
```

这就是整套坐标变换链条——相机标定提供 "像素→相机坐标系" 的钥匙，手眼标定提供 "相机→云台" 的钥匙，IMU 提供 "云台→世界" 的钥匙。三把钥匙凑齐，一个像素就能定位到世界中的真实位置。
